z-index

三维坐标系理解：

  • x轴：水平方向（左右）
  • y轴：垂直方向（上下）
  • z轴：垂直于屏幕的方向（前后）

其实说白了，就是控制优先级，这么理解，优先级越高，就在最上面，也可以叫：堆叠上下文

取值说明：

  • auto：默认值，堆叠顺序与父元素相同
  • number：整数（正数、负数、0）
  • inherit：继承父元素的 z-index

z-index 生效的前提条件

重要：z-index 只对以下定位元素生效：

  • position: relative
  • position: absolute
  • position: fixed
  • position: sticky

overflow

规定了当元素的内容超出其指定的区域时，应该如何处理。

overflow: hidden; 的意思是：将溢出的内容直接裁剪掉，并且不可见，不提供任何滚动机制。

为了更好地理解 hidden，我们看一下 overflow 的其他常用值：

属性值 作用
overflow: visible 默认值。内容不会被裁剪，会原样呈现在元素框之外。
overflow: hidden 裁剪溢出的内容，不可滚动查看。
overflow: scroll 总是显示滚动条（即使内容没有溢出）。用户可以通过滚动查看被隐藏的内容。
overflow: auto 智能。由浏览器决定，如果内容溢出，则显示滚动条；如果没有溢出，则不显示。这是最常用的值之一。

你可以使用 overflow-x 和 overflow-y 来分别控制水平和垂直方向的溢出行为（例如 overflow-x: hidden; overflow-y: auto;）。

触发BCF

BFC 是什么？
你可以把它理解为一个独立的布局容器，容器内部的布局不会影响到外部，外部的布局也不会影响到内部。

触发 BFC 能解决什么问题？

  1. 外边距合并
    在普通流中，相邻两个块级元素的上下外边距会发生合并。但如果其中一个元素被包裹在一个触发了 BFC 的容器内，它们的边距就不会合并。

  2. 阻止元素被浮动元素覆盖
    如果一个元素旁边有一个浮动元素，它可能会被浮动元素覆盖。通过给这个元素设置 overflow: hidden 触发 BFC，它会避开浮动元素，形成环绕效果。

background-attachment

是 CSS 中用于控制背景图像滚动行为的属性，它决定了背景图像相对于视口或包含元素的固定方式。

属性值及效果

1**. scroll（默认值）**

  • 效果：背景图像会随着页面的滚动而滚动
  • 特点：
    • 背景图像相对于元素本身固定
    • 当元素内容滚动时，背景图像会一起滚动
    • 这是最常用的默认行为

2. fixed

  • 效果：背景图像相对于视口固定，不随页面滚动
  • 特点：
    • 创建视差滚动效果
    • 背景图像始终停留在屏幕的同一位置
    • 性能开销较大，特别是在移动设备上

3. local

  • 效果：背景图像随元素内容的滚动而滚动
  • 特点：
    • 当元素有滚动条时，背景图像会随内容一起滚动
    • 适用于可滚动容器内的背景
